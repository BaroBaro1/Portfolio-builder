# Project Principles

## 1. Simplicity First

Always choose the simplest solution that solves the problem.

---

## 2. Finish Before Expanding

Version 1 must be completed before adding any new feature.

New ideas go to Backlog.md.

---

## 3. Learn by Building

The project is the learning path.

No long tutorials before implementation.

---

## 4. AI is a Team Member

AI helps build the project.

AI does not make product decisions.

---

## 5. Understand Before Accepting

Never accept generated code without understanding its purpose.

---

## 6. Reusable Architecture

Build reusable components.

Avoid duplicated code.

---

## 7. User Experience First

Every decision should improve the user's experience.

---

## 8. Documentation Before Implementation

Important decisions are documented before writing code.
